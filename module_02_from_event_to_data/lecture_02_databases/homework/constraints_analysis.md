## Перечень ограничений (Constraints)

* Для таблицы Sales:

NOT NULL: sales_id, product_id, total_price. Обоснование: Мы не можем совершить продажу без идентификатора, без самого товара или без указания цены.

CHECK: quantity > 0 (нельзя продать 0 или минус товаров) и discount >= 0 AND discount <= 100 (скидка не может быть отрицательной или более 100%).

UNIQUE: sales_id (первичный ключ по определению уникален).

* Для таблицы Products:

NOT NULL: product_id, product_name, price. Обоснование: Товар в каталоге бесполезен без названия и цены.

CHECK: price >= 0 (цена не может быть отрицательной).

UNIQUE: product_name (чтобы избежать создания дубликатов одного и того же товара с разными ID).
