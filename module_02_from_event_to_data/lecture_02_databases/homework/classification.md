## Классификация данных магазина продуктов
В ходе анализа предоставленных файлов была проведена классификация данных на две основные категории: Мастер-данные (статичные справочники) и Транзакционные данные (факты событий).

## 1. Транзакционные данные (Факты)
Файл sales.csv (Сущность: Продажа): Это центральный файл системы. В нем фиксируются динамические события — факты совершения покупок. Данные обновляются при каждом пробитом чеке и содержат ссылки на другие таблицы (кто купил, что купил и кто продал).

## 2. Мастер-данные (Сущности и справочники)
Это относительно стабильные данные, которые описывают участников и объекты бизнес-процесса:

Файл products.csv (Сущность: Товар): Основной каталог продукции. Содержит информацию о наименованиях, ценах и характеристиках товаров, которые магазин предлагает клиентам.

Файл employees.csv (Сущность: Сотрудник): Реестр персонала магазина. Хранит персональные данные сотрудников и информацию об их трудоустройстве.

Файл customers.csv (Сущность: Клиент): База данных зарегистрированных покупателей. Позволяет идентифицировать участников программы лояльности при совершении сделок.

Файл shops.csv (Сущность: Магазин): Список физических торговых точек с указанием их точных адресов.

Файл categories.csv (Сущность: Категория): Вспомогательный справочник для логической группировки товаров.

Файл cities.csv (Сущность: Город): Географический справочник, используемый для связи магазинов и клиентов с конкретными населенными пунктами.

Файл countries.csv (Сущность: Страна): Глобальный справочник стран для определения регионального присутствия торговой сети.
