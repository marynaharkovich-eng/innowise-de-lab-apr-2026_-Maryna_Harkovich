## Перечень сущностей и их первичных ключей (PK)
В реляционной базе данных Primary Key (PK) — это столбец, который уникально идентифицирует каждую запись и не может быть NULL.
Сущность      Первичный ключ (PK)     Обоснование выборa
Sales         sales_id                Уникальный номер каждой транзакции.
Categories    category_id             Уникальный код категории для связи с товарами.
Employees     employee_id             Личный номер сотрудника в реестре компании.
Customers     customer_id             Уникальный ID клиента в программе лояльности.
Cities        city_id                 Уникальный идентификатор населенного пункта.
Products      product_id              Уникальный артикул товара в каталоге.
Shops         shop_id                 Уникальный номер торговой точки.
Countries     country_id              Уникальный код страны.

## Для сущности Customers выбран customer_id, так как имя или адрес могут повторяться у разных людей, а ID гарантирует уникальность каждой записи. Для Categories выбран category_id, так как названия категорий теоретически могут измениться, но их цифровой код в системе останется стабильным и уникальным.
