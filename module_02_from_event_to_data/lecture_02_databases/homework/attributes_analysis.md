## Анализ атрибутов (Products и Employees)
## Список атрибутов и проверка на атомарность:
## Сущность Products:

* product_id: Атомарный. Уникальное целое число.

* product_name: Атомарный. Содержит только название конкретной позиции.

* price: Атомарный. Одиночное числовое значение стоимости.

* category_id: Атомарный. Ссылка на одну категорию.

* vitality_days: Атомарный. Одно число, указывающее срок хранения.

(Остальные атрибуты class, modify_timestamp, resistant, is_allergic также атомарны, так как содержат по одному значению/флагу на ячейку).

## Сущность Employees:

* employee_id: Атомарный. Уникальный идентификатор.

* first_name / last_name: Атомарные. Имя и фамилия разделены по разным колонкам, что является классическим примером соблюдения атомарности (в отличие от хранения "ФИО" одной строкой).

* middle_initial: Атомарный. Содержит только один символ инициала.

* birth_date / hire_date: Атомарные. Содержат одиночные значения даты.

* city_id / shop_id: Атомарные. Ссылки на конкретные объекты.

## Обоснование: Все вышеперечисленные атрибуты являются атомарными, так как не содержат массивов, вложенных JSON-объектов или списков, разделенных запятыми. Каждая ячейка представляет собой минимальную смысловую единицу данных, необходимую для работы системы.
